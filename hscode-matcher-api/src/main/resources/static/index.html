Options de test (lab)

Ces réglages s’appliquent aux appels GET /api/v1/search depuis cette page. Les cases et champs vides laissent le comportement par défaut du serveur.

Si décoché, chaque recherche envoie hybrid=false : pas d’embedding de la requête ni fusion RRF — uniquement la recherche lexicale (Lucene). Utile pour comparer avec le mode sémantique.

Hybrid côté client (longueur minimale)

Tant que la requête (après espaces en trop retirés) a moins de N caractères, la page envoie hybrid=false même si la case hybrid est cochée — pour éviter l’embedding sur des toutes petites saisies. 0 = pas de garde côté navigateur (seul le serveur peut encore couper via seuil hybrid serveur).

Limite résultats API

Paramètre limit : nombre maximum de codes renvoyés dans la réponse (plafonné à 50 par l’API). Plus la valeur est élevée, plus le calcul (et le JSON) peuvent être lourds.

Active les requêtes fuzzy sur les tokens (tolérance aux fautes de frappe) en plus du BM25. Décoché → fuzzy=false : seule la partie « plein texte » Lucene reste.

Longueur minimale des tokens pour le fuzzy

Paramètre minFuzzyTokenLength : un token analysé plus court que cette valeur ne reçoit pas de clause fuzzy. Champ vide ou 0 → défaut serveur (3). Valeurs possibles 2–32. Plus le seuil est haut, moins il y a de fuzzy (moins de bruit, moins de rappel).

Envoie explain=true. La réponse JSON contient alors les rangs lexical et sémantique (avant fusion) et les lignes de fusion RRF — pour comprendre d’où viennent les codes affichés. Données plafonnées côté serveur.

Seuil hybrid côté serveur (caractères)

Paramètre minHybridChars : l’API refuse l’hybrid (embedding + RRF) si la requête tronquée fait moins de N caractères. Champ vide → valeur de hs.matcher.hybrid.min-query-chars dans application.properties. 0 = pas de seuil longueur côté API pour cet appel. Visible dans debug.serverMinHybridQueryChars.

Timeout embedding (ms)

Paramètre embedTimeoutMs : durée max pour calculer le vecteur de la requête ONNX. Dépassement → retour lexical, debug.embeddingTimedOut, compteur métrique hs.matcher.embed (outcome=timeout). Vide → hs.matcher.embed.timeout-ms (0 = pas de timeout).

Debounce (ms)

Délai après la dernière frappe avant d’appeler l’API. Réduit le nombre de requêtes pendant la saisie rapide ; un délai plus long rend l’interface plus « calme » mais moins réactive.

RRF k

Paramètre rrfK : constante k de la fusion reciprocal rank fusion entre liste lexicale et liste sémantique. Plus k est grand, plus les rangs profonds comptent faiblement. Vide → hs.matcher.hybrid.rrf-k. Réponse : effectiveRrfK.

Multiplicateur pool lexical

Paramètre poolMultiplier : taille du pool de candidats (lexical puis sémantique) avant la fusion, dérivée de limit × multiplicateur, plafonnée côté serveur (200). Plus le pool est grand, plus le hybrid voit de codes mais le coût augmente. Vide → hs.matcher.hybrid.lexical-pool-multiplier.

Lance deux requêtes en parallèle avec les mêmes autres paramètres : une avec hybrid autorisé, une en lexical pur. Affiche deux colonnes pour comparer les classements. La liste de suggestions sous le champ utilise toujours le mode « auto » (hybrid selon les cases ci-dessus).
